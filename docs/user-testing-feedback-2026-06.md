# 银龄智护两周用户测试反馈整理

## 测试概况

- 测试周期：两周
- 测试对象：5 位老年用户
- 年龄范围：69 岁至 81 岁
- 主要特征：轻度白内障、低视力、行动较慢、膝盖不便、糖尿病用药管理、独居或与配偶同住
- 测试重点：巡路、找物、跌倒确认、语音交互、管理端记录、隐私感知和日常使用负担

## 核心结论

1. 语音优先和“行动型引导”被明确认可。
   用户认为“先停下”“往右一点”“扶着门框”“脚尖先探一下”这类可执行指令，比单纯描述画面更适合视力不佳老人。

2. 找物功能有实际价值，但需要增强小物体和低对比目标识别。
   老人能够用它减少呼叫家人的次数，尤其在桌面杂乱场景下有帮助。主要问题是目标太小、颜色接近背景时回答含糊。

3. 跌倒确认流程方向正确，但紧急状态下的取消和确认必须更明显。
   用户认可“先询问再倒计时报警”，认为不会因为手机掉落就立刻惊扰家属。问题是电视声或紧张状态下可能听不清、反应不过来。

4. 管理端记录能降低家庭沟通成本，但隐私说明必须前置。
   用户认可用药、风险和事件记录同步给家属，但明确要求知道哪些数据会上传、谁能看、能否关闭。

5. 日常模式需要更简单，设置页应主要由家属完成。
   老人日常只需要几个大按钮和语音操作。设置页对低视力和高龄用户仍偏复杂，应增加“家属设置模式”和“老人日常模式”。

6. 语音链路需要继续优化。
   主要问题包括播报稍慢、提示过长、用户说话慢时提前响应、方言识别不好、手机必须手持带来的疲劳。

## 产品改造清单

### P0：必须优先修复

1. 紧急跌倒确认语音增强
   - 检测到疑似跌倒后，确认提示使用更短句式。
   - 播报时提升音量，配合震动。
   - 明确支持固定语音口令：“没事”“取消报警”“我没有摔倒”。
   - 设置页展示并可测试取消报警口令。

2. 语音播报和自动刷新节奏优化
   - 自动模式下，下一次导航刷新应等待上一段关键语音播报完成或进入安全截断点。
   - 首句必须短，优先输出“先停下”“可以走”“不要跨”等安全结论。
   - 长说明放在第二句，不阻塞安全提示。

3. 老人日常模式与家属设置模式分离
   - 老人首页只保留巡路、找物、呼叫、设置入口。
   - 复杂模型、云端、本地、ASR、TTS、隐私设置默认放入家属设置模式。
   - 设置项增加语音说明和大字号。

4. 隐私中心
   - 明确展示：哪些数据本地处理、哪些数据会上传、谁可以查看、保留多久。
   - 提供一键本地优先、一键云端优先。
   - 照护人员访问记录可追踪。

### P1：影响体验和准确率

1. 小物体找物增强
   - 当目标小或低对比时，不直接给确定结论。
   - 引导用户靠近、开灯、换角度、将手机对准桌面。
   - 输出置信度和下一步动作：“我不确定，先把手机向右转一点”。

2. 慢速语音和方言适配
   - 增加慢速说话模式，延长端点检测等待时间。
   - ASR 支持本地和云端切换，并提供测试句。
   - 对短指令建立白名单，例如“停”“没事”“取消”“找药盒”。

3. 硬件使用负担降低
   - 增加挂脖、胸前支架、桌面支架的推荐角度说明。
   - 增加“支架模式”，默认降低操作频率，以语音唤醒为主。

### P2：增强竞争力

1. 用药和长护记录 Agent
   - 支持用户通过对话记录喝水、用药、疼痛、头晕、睡眠等状态。
   - 管理端形成趋势和异常提醒。

2. 家属复核闭环
   - 风险事件、跌倒误报、人工修正都生成可复核记录。
   - 家属可标记“已确认”“误报”“需回访”。

3. 用户研究持续量化
   - 每次测试记录任务完成率、平均响应时间、误报率、免看屏完成率、照护复核率。
   - 将主观反馈和自动化 benchmark 结果对应起来。

## 可写入比赛材料的表述

本项目已完成 5 位老年用户、连续两周的真实使用反馈收集。测试对象覆盖轻度白内障、低视力、行动较慢、膝盖不便、糖尿病用药管理、独居和与配偶同住等典型居家长护场景。反馈显示，用户最认可的是“语音优先”和“行动型引导”：系统不是单纯描述画面，而是直接给出“先停下”“扶住门框”“向右一点”等可执行提示。找物、跌倒确认和家属管理端均体现出明确使用价值，同时测试也暴露出语音延迟、播报长度、慢速语音识别、方言适配、隐私透明度、手持负担和设置复杂度等改进方向。

基于本轮测试，项目后续将重点优化四项能力：第一，紧急跌倒确认和取消报警的语音、震动和按钮提示；第二，自动巡路中的短句优先播报和安全提示优先级；第三，老人日常模式与家属设置模式分离；第四，隐私中心和照护端复核闭环。上述改造将进一步提升低视力和行动不便老人独自在家活动时的可用性、安全感和家庭协同效率。
